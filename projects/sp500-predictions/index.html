S&P 500 Predictions MACHINE LEARNING  ·  TIME SERIES  ·  FINANCE

Abstract

This project builds an end-to-end predictive pipeline for the S&P 500 index using the Goyal-Welch (2022) dataset spanning nearly a century of U.S. market data. Starting from raw macroeconomic and financial predictors, the analysis covers data preprocessing, two rounds of feature selection (Lasso, Ridge, Boruta), lag identification via VAR and ARIMA models, feature engineering, and the training and evaluation of four machine learning models: a simplified Keras neural network, a Keras neural network with multi-start optimization, a Keras neural network with hyperparameter tuning, and a Gaussian Process. The best-performing model achieves an out-of-sample RMSE of 88.70.

Predicting the S&P 500: A Machine Learning Approach

by Mathieu Breier

May 21, 2024  ·  GitHub Repository

1. Introduction

Can decades of macroeconomic data tell us something meaningful about where the stock market is heading? That question sits at the heart of this project.

Predicting the S&P 500 stock market index has long been one of the most studied — and most humbling — problems in empirical finance. Markets are noisy, driven by sentiment, geopolitical shocks, and forces that no model can fully anticipate. And yet, a rich body of academic research suggests that certain financial and macroeconomic variables do carry predictive signal, at least over longer horizons. Understanding which variables matter, how to extract that signal, and which methods are best suited to do so is both a practical and intellectually fascinating challenge.

This project was born out of a desire to learn by doing. Rather than reviewing methods in the abstract, the goal was to build a full, end-to-end predictive pipeline — from raw data to model evaluation — using real financial data and a diverse set of machine learning techniques. It is also intended to serve as a teaching resource: each step is documented and explained so that anyone curious about applied machine learning in finance can follow along, understand the reasoning behind each decision, and adapt the methods to their own questions.

The analysis is built around the Goyal-Welch (2022) monthly dataset, a widely used benchmark in the return predictability literature that aggregates a broad set of macroeconomic and financial predictors spanning nearly a century of U.S. market data (November 1926 to December 2021). From this dataset, a range of financial indicators are constructed — including log returns, the dividend-price ratio, earnings yield, dividend yield, the default yield spread, and the log equity premium — which serve as the building blocks of the predictive models.

The analysis follows a structured, step-by-step pipeline designed to be both rigorous and transparent:

  1. Data Cleaning & Preprocessing — raw data is cleaned, transformed, and enriched with derived financial indicators.
  2. Exploratory Data Analysis — time series plots, return distributions, and a correlation heatmap are used to uncover structure and relationships in the data.
  3. Feature Selection (Round 1) — Lasso, Ridge, and Boruta (Random Forest-based) methods are applied to identify the most relevant predictors from the original variable set.
  4. Lag Identification — VAR model selection criteria and individual ARIMA models are used to determine the optimal number of lags for each variable.
  5. Feature Engineering & Selection (Round 2) — lagged versions of the selected features are created, and a second round of feature selection is performed to retain only the most informative lags.
  6. Predictive Modeling — multiple machine learning models are trained and evaluated on the final feature set to forecast the S&P 500 index.

2. Data

2.1 Dataset

The raw data comes from the Goyal-Welch (2022) monthly dataset, a standard benchmark in the empirical return predictability literature. The dataset covers November 1926 to December 2021 (1,141 monthly observations) and contains a broad set of financial and macroeconomic series, including the S&P 500 index level, dividends, earnings, bond yields, and various other indicators.

After loading the data, the Index column is converted to numeric (removing thousands separators), dates are parsed into a proper monthly date format, and the dataset is filtered to the November 1926–December 2021 window. A first row used solely for return calculation is removed, leaving a clean working dataset of 1,140 observations.

2.3 Missing Values

After construction, only the variable csp (cross-sectional premium) contains missing values. These are imputed with the variable's sample mean, a standard approach for series where the number of missing observations is small relative to the total sample.

4. Feature Selection I

This section applies three complementary methods to identify the most predictive variables from the original feature set before any lag construction. The goal is to reduce the dimensionality of the problem and remove variables that contribute little or no signal.

4.1 Lasso Regression

Lasso (Least Absolute Shrinkage and Selection Operator) adds an \(\ell_1\) penalty to the ordinary least-squares objective, which can shrink some coefficients to exactly zero, effectively removing the corresponding variables from the model. It is therefore well-suited as a feature selection tool.

\[ \hat{\beta}^{\text{Lasso}} = \arg\min_{\beta} \left\{ \|y - X\beta\|_2^2 + \lambda \|\beta\|_1 \right\} \]

The regularization parameter \(\lambda\) is tuned by cross-validation. All features are standardized prior to fitting (mean zero, unit variance) to ensure fair penalization. The Lasso identifies IndexDiv (the sum of the S&P 500 index and dividends) as the dominant predictor. This is intuitive: IndexDiv already incorporates the index level, making it a near-perfect linear proxy for the target.

4.2 Ridge Regression

Ridge Regression penalizes the squared magnitude of the coefficients (\(\ell_2\) penalty). Unlike Lasso, it does not shrink any coefficient to exactly zero, but it reduces the influence of less important features, particularly under multicollinearity.

\[ \hat{\beta}^{\text{Ridge}} = \arg\min_{\beta} \left\{ \|y - X\beta\|_2^2 + \lambda \|\beta\|_2^2 \right\} \]

Ridge regression confirms IndexDiv as the most significant predictor. E12 (earnings) and D12 (dividends) also carry large positive coefficients (72.65 and 58.76 respectively), consistent with the idea that profitability and dividend payments are key drivers of the index. The book-to-market ratio (b.m) adds further explanatory power, suggesting that valuation multiples matter beyond pure earnings and dividend levels.

4.3 Boruta (Random Forest-based)

The Boruta algorithm provides a complementary, non-parametric perspective on feature importance. It works by creating shadow features — randomly permuted copies of each original feature — and iteratively comparing the importance of original features against these uninformative shadows using a Random Forest. A feature is retained only if it consistently outperforms the best shadow feature. This process makes Boruta particularly robust to interactions and non-linearities that linear methods may miss.

Boruta identifies five confirmed features in descending order of mean importance:

  1. IndexDiv — mean importance 16.03 (highest; consistent with Lasso and Ridge)
  2. D12 — trailing 12-month dividends
  3. E12 — trailing 12-month earnings
  4. BAA — BAA-rated corporate bond yield
  5. lty — long-term government bond yield
All three methods consistently highlight IndexDiv, D12, and E12 as the most informative predictors. The inclusion of bond yield variables (BAA, lty) further suggests that interest rate conditions carry meaningful signal for equity market levels.

5. Lag Identification

Financial time series exhibit autocorrelation: past values of a variable often carry information about its future. To capture this temporal structure, I determine the optimal number of lags to include for each variable using two approaches.

5.1 Method 1: VAR Model Selection (HQ Criterion)

A Vector Autoregressive (VAR) model treats all variables simultaneously and estimates the lag structure that best fits the joint system. The Hannan-Quinn (HQ) information criterion is used to select the optimal lag length \(p\):

\[ \text{HQ}(p) = \ln|\hat{\Sigma}_p| + \frac{2\ln(\ln T)}{T} \cdot k^2 p \]

where \(\hat{\Sigma}_p\) is the residual covariance matrix, \(T\) is the sample size, and \(k\) is the number of variables. The HQ criterion penalizes model complexity more aggressively than AIC but less than BIC. Searching up to a maximum of 15 lags, the HQ criterion selects the global optimal lag order for the joint system of variables.

5.2 Method 2: ARIMA per Variable

Since the VAR approach finds a single lag length for all variables jointly, it may not be optimal for each series individually. As a complement, auto.arima() is applied to each variable separately, automatically selecting the best ARIMA(\(p, d, q\)) model by minimizing an information criterion (AIC/BIC). The AR order \(p\) from each fitted model gives the variable-specific optimal lag.

Based on both methods, I proceed to construct up to 10 lags for the selected variables, providing ample temporal depth for the subsequent feature selection step to determine which lags are truly informative.

6. Feature Engineering

Based on the results of Feature Selection I, the following 12 variables are selected for lag expansion: SP500, IndexDiv, D12, E12, BAA, lty, AAA, dy, dp, ep, ntis, b.m. For each variable, lags 1 through 10 are created, yielding a total of 120 lagged features plus the contemporaneous values (before removing the original un-lagged SP500).

Formally, for variable \(x_t\), the lag-\(k\) feature is defined as:

\[ x_{t-k}, \quad k = 1, 2, \ldots, 10 \]

Rows containing any missing values (arising at the beginning of the sample from the lag construction) are removed, and the original SP500 series is retained only as the target variable \(y_t\). The final feature matrix is then merged with the target to form the working dataset for the second round of feature selection.

7. Feature Selection II

With 120+ lagged features in hand, a second round of feature selection is performed to identify which lags carry genuine predictive content and which are redundant. The same three methods — Lasso, Ridge, and Boruta — are applied to this expanded feature set.

All three methods converge on the same conclusion: the vast majority of lags carry minimal additional information once the two most informative lagged features are included. The final feature set is reduced to just two variables:

  • SP500_lag2: the S&P 500 index two months prior
  • IndexDiv_lag1: the index-plus-dividends value one month prior
The parsimony of this final feature set — just two lagged predictors — has an important side-benefit: with only two features, the loss function of the neural network can be visualized as a 3D surface, providing direct insight into the optimization landscape explored during training.

8. Neural Network Models

8.1 Data Preparation & Standardization

Before training, all features and the target variable are standardized to have mean zero and unit variance:

\[ \tilde{x} = \frac{x - \bar{x}}{s_x} \]

Standardization is critical for neural networks: without it, features with larger magnitudes dominate the gradient descent updates, causing erratic optimization. After training, predictions are de-standardized back to the original scale before computing evaluation metrics.

The dataset is split 80/20 by time (not randomly) to respect the temporal ordering of the data: the training set covers approximately 1927–2003, and the test set covers 2003–2021.

8.2 Architecture

All three neural network variants share the same base architecture: a fully connected feedforward network with two hidden layers using the ReLU activation function, and a single linear output unit for regression.

\[ \text{ReLU}(z) = \max(0,\, z) \]

The network is trained to minimize the mean squared error (MSE) loss:

\[ \mathcal{L} = \frac{1}{n} \sum_{i=1}^{n} \left(\hat{y}_i - y_i\right)^2 \]

Performance is reported as the root mean squared error (RMSE) on the test set, computed on de-standardized predictions:

\[ \text{RMSE} = \sqrt{\frac{1}{n} \sum_{i=1}^{n} \left(\hat{y}_i - y_i\right)^2} \]

8.3 Model I — Simplified Neural Network

The first model uses a straightforward architecture: two hidden layers of 256 and 128 units respectively, trained with the Adam optimizer (learning rate 0.0005) and early stopping (patience 30 epochs, monitoring validation loss). This serves as the baseline against which more complex variants are compared.

8.4 Model II — Multi-Start Optimization

Neural networks are non-convex optimizers: gradient descent may converge to different local minima depending on the initial weights. A multi-start strategy trains the model 10 times from independently random initializations and selects the run that achieves the lowest validation loss. This reduces sensitivity to a poor initialization and provides a more reliable estimate of what the architecture can achieve.

[Figure: Training Histories for 10 Multi-Start Runs]

Figure 5 — Training and validation loss curves for each of the 10 random initializations. The best run (lowest validation loss) is retained as the final model.

8.5 Model III — Hyperparameter Tuning with Regularization

The third variant adds two forms of regularization to the base architecture and conducts a grid search over key hyperparameters.

Dropout randomly sets a fraction \(\delta\) of hidden unit activations to zero at each training step, preventing co-adaptation of neurons and reducing overfitting:

\[ h_i^{\text{drop}} = h_i \cdot \text{Bernoulli}(1 - \delta) \]

L2 kernel regularization penalizes large weight values, analogous to Ridge regression on the network weights:

\[ \mathcal{L}_{\text{reg}} = \mathcal{L} + \alpha \sum_{l} \|W_l\|_2^2 \]

The grid search explores the following hyperparameter space:

  • Units per layer: [32–16], [64–32], [128–64], [256–128], [512–256]
  • Dropout rate: 0.1, 0.3, 0.5
  • Learning rate: 0.0001, 0.0002, 0.0003, 0.0005
  • Batch size: 8, 16, 32

The optimal configuration found is: [512, 256] units, dropout 0.3, learning rate 0.0005, batch size 8. The final model is retrained from scratch with these parameters and early stopping.

9. Gaussian Process Regression

A Gaussian Process (GP) is a non-parametric probabilistic model that places a prior distribution over functions. For regression, the posterior predictive distribution at a new input \(x^*\) is:

\[ f(x^*) \mid X, y \sim \mathcal{N}\!\left(\mu^*(x^*),\; \sigma^{*2}(x^*)\right) \]

where the mean and variance are determined by the kernel function \(k(x, x')\), which encodes assumptions about the smoothness and structure of the underlying function.

Since the S&P 500 index is non-stationary — its statistical properties change over time, and its level grows persistently — a standard stationary kernel (such as the RBF/Gaussian kernel) is inappropriate. I instead define a custom non-stationary kernel that combines a stationary decay term with a linear inner product:

\[ k(x, y) = 2\exp\!\left(\frac{-\|x - y\|_1}{2 \times 1.5^2}\right) + 1.5\, x^\top y \]

The first term captures local similarity between observations (similar to an exponential kernel), while the second term — the linear kernel — allows the model to extrapolate the long-term trend in index levels. The noise variance is set to \(\sigma^2_n = 0.003\).

The GP is fitted on the standardized training set and predictions are de-standardized before RMSE computation, consistent with the neural network evaluation protocol.

11. Conclusion

This project set out to predict the S&P 500 index using a variety of machine learning models. Multiple approaches were explored, including neural networks with different architectures and optimization techniques, as well as Gaussian Process Regression. The results provide valuable insights into the strengths and limitations of each method and highlight the importance of model complexity, parameter optimization, and feature engineering in time series forecasting.

Models Performance Overview

Simple Neural Network. This model used a basic architecture with two hidden layers of moderate complexity, trained with early stopping. The RMSE was relatively high, reflecting the limitations of a plain architecture in capturing the complexity of S&P 500 dynamics, which are influenced by a wide range of economic factors.

Neural Network with Multi-Start Optimization. The multi-start approach trained the model multiple times from different random initializations, avoiding poor local minima. This produced a modest improvement in RMSE, confirming that better initialization conditions improve model generalizability.

Neural Network with Hyperparameter Tuning. This model used a deeper architecture with dropout regularization and L2 kernel regularization, with hyperparameters optimized through grid search. It achieved the lowest RMSE among all models, demonstrating that a carefully tuned, more sophisticated architecture better captures the patterns in the data.

Gaussian Process Regression. Applied with a custom non-stationary kernel to capture the non-linearities and trend of the stock market, the GP model had a slightly higher RMSE compared to the best neural network. Nevertheless, it offers a valuable advantage: unlike neural networks, Gaussian Processes provide uncertainty estimates on predictions, which are crucial in financial contexts where quantifying risk is as important as the point forecast itself.

Key Takeaways

  • Feature engineering matters. Proper feature selection and lag construction played a pivotal role in model performance. Despite starting with over 120 candidate features, the final model relied on just two: SP500_lag2 and IndexDiv_lag1. Methods like Lasso, Ridge, and Boruta were effective in identifying this minimal informative set.
  • Optimization strategy matters. Multi-start optimization demonstrated the value of avoiding poor local minima. Hyperparameter tuning showed that architecture choices — number of units, dropout rate, learning rate, batch size — have a significant impact on generalization.
  • Regularization matters. Dropout and L2 regularization reduced overfitting, allowing more complex models to generalize better on the 20-year test window.
  • Model choice is context-dependent. While deep neural networks achieved the best point forecast accuracy, Gaussian Processes offer interpretable uncertainty bounds that can be more valuable than marginal RMSE improvements in a risk-aware financial application.

For future work, incorporating additional macroeconomic indicators and applying sequence models such as Long Short-Term Memory (LSTM) networks could further improve prediction accuracy. An ensemble approach combining the predictions of multiple models may also improve robustness by averaging out individual model errors.
